# 工伤理赔试算平衡工具使用说明

这是一个用于测算工伤理赔对企业未来工伤保险费率影响的网页工具。工具会根据企业行业风险类别、当前费率浮动状态、缴费基数和本次预计报销金额，推演未来三年在“正常走工伤报销”和“企业内部解决”两种情况下的保费差额，并给出财务视角下的建议。

## 一、工具适用场景

适合用于以下场景：

- 企业发生工伤事故后，想快速评估“走工伤报销”是否会导致未来保费明显增加。
- 企业当前已享受工伤保险费率下浮优惠，需要判断一次理赔是否会清零优惠并带来更高成本。
- 人事、社保、财务或安全管理人员需要向管理层说明工伤理赔对未来三年成本的影响。
- 想根据重庆市工伤保险费率浮动规则，做一个初步的内部测算和决策参考。

本工具不是法律意见或最终社保核定结果，只用于辅助测算。实际费率、待遇支付和政策适用，应以当地社保经办机构最终核定为准。

## 二、在线使用

打开项目部署后的网页地址即可使用。

如果你是从本仓库本地运行，默认地址为：

```text
http://127.0.0.1:5004
```

页面打开后不需要登录，也不需要上传文件，直接填写页面左侧或移动端上方的参数即可。

## 三、本地运行方法

### 1. 安装 Python

建议使用 Python 3.10 或更高版本。

检查 Python 是否可用：

```bash
python --version
```

### 2. 安装依赖

进入项目目录后执行：

```bash
python -m pip install -r requirements.txt
```

项目依赖很少，主要包括：

- `flask`：用于本地网页服务。
- `gunicorn`：用于线上部署环境。

### 3. 启动服务

```bash
python app.py
```

启动成功后访问：

```text
http://127.0.0.1:5004
```

如果端口 5004 被占用，可以修改 [app.py](app.py) 中的端口号：

```python
app.run(host='0.0.0.0', port=5004, debug=False)
```

## 四、页面字段说明

### 1. 行业工伤风险类别

选择企业所属的工伤风险类别。页面内置一类至八类行业基准费率。

如果不清楚企业属于哪一类，可以点击行业类别右侧的放大镜按钮，打开“行业工伤风险类别对照表”。在弹窗中点击某一行，即可自动选中对应类别。

类别与基准费率大致如下：

| 类别 | 基准费率 |
| --- | --- |
| 一类 | 0.30% |
| 二类 | 0.60% |
| 三类 | 0.90% |
| 四类 | 1.10% |
| 五类 | 1.30% |
| 六类 | 1.60% |
| 七类 | 1.90% |
| 八类 | 2.20% |

注意：一类行业通常不参与下浮测算，因此选择一类后，“当前费率浮动状态”的可选项会自动切换。

### 2. 缴费基数总额（万元/月）

填写企业每月工伤保险缴费基数总额，单位是“万元/月”。

示例：

- 如果企业月缴费基数总额为 1000 万元，填写 `1000`。
- 工具会自动计算年度基数，即 `月缴费基数 × 12`。

这个字段会直接影响年度保费：

```text
年度保费 = 月缴费基数 × 12 × 行业基准费率 × 当前浮动档次
```

### 3. 当前费率浮动状态

选择企业当前处于哪个费率档次。

二至八类行业常见选项包括：

- 连续 3 年无理赔，下浮至 50%。
- 连续 2 年无理赔，下浮至 80%，明年可能降至 50%。
- 连续 1 年无理赔，下浮至 80%。
- 去年有理赔，当前执行基准档 100%。
- 去年理赔较多，当前上浮至 120%。
- 去年理赔极多，当前顶格上浮至 150%。

这个状态是未来三年推演的起点。选错该字段，会明显影响测算结果。

### 4. 预估社保可报销总额（万元）

填写本次工伤预计可以由工伤保险基金支付或报销的金额，单位是“万元”。

示例：

- 预计可报销 2 万元，填写 `2`。
- 预计可报销 15.5 万元，填写 `15.5`。

工具会将该金额与当前年度工伤保险缴费额进行比较，计算支缴率：

```text
支缴率 = 本次预计报销金额 ÷ 本年已缴保费预估
```

支缴率越高，越可能导致次年费率上浮。

### 5. 触犯“一票否决”红线

如果本次事故涉及以下情况之一，需要勾选该项：

- 发生生产安全死亡责任事故，死亡 2 人及以上。
- 被安全生产监督管理部门列入安全生产黑名单。

勾选后，工具会按次年费率强制上浮至 150% 进行测算。

## 五、结果怎么看

页面右侧或移动端结果区域会给出核心建议。

### 1. 建议企业内部解决

当页面显示“建议企业内部解决”时，说明：

- 本次工伤报销金额虽然能带来短期收入；
- 但会推高支缴率，影响未来费率档次；
- 未来三年额外增加的工伤保险费高于本次可报销金额；
- 从账面测算看，走工伤报销反而可能产生净亏损。

此时页面会展示：

```text
社保报销收入
三年额外多缴保费
账面净亏损
```

### 2. 建议正常走工伤报销程序

当页面显示“建议正常走工伤报销程序”时，说明：

- 本次可报销金额高于未来三年预计增加的保费；
- 即使后续费率有阶段性影响，综合测算仍然更适合走工伤报销；
- 从账面测算看，正常报销更划算。

此时页面会展示：

```text
社保报销收入
三年额外多缴保费
账面净节省
```

### 3. 本年已缴保费预估

显示企业在当前费率状态下，本年度预计缴纳的工伤保险费。

它用于计算本次理赔对支缴率的影响。

### 4. 推高支缴率至

显示本次报销后，预计形成的支缴率。

常见规则理解：

- 支缴率为 0：可能享受下浮优惠。
- 支缴率低于 100%：通常执行基准档。
- 支缴率达到或超过 100%：可能进入上浮档。
- 支缴率达到或超过 120%：可能进入顶格上浮档。

### 5. 三年后财务净变化

显示从三年合计角度看，走工伤报销相对于内部解决的账面变化。

- 正数：报销收益大于额外保费。
- 负数：额外保费大于报销收益。

## 六、未来三年保费推演明细

表格会按年份展示两条路径：

### 如果选择内部解决

表示本次事故不计入工伤保险基金支付，企业继续按“零理赔”路径自然恢复或维持下浮档次。

### 如果走工伤报销

表示本次事故进入工伤保险基金支付，支缴率上升，未来费率按照对应规则变化。

### 多缴成本

每一年“走工伤报销”相比“内部解决”多出来的保费。

三年累计额外缴纳保费总额，是本工具判断建议方向的关键依据。

## 七、手机端使用建议

本工具已适配手机端。

手机端使用时建议按以下顺序操作：

1. 先选择行业工伤风险类别。
2. 输入缴费基数总额。
3. 选择当前费率浮动状态。
4. 输入本次预计可报销金额。
5. 如有红线情形，勾选“一票否决”。
6. 查看底部固定结果条，快速判断当前建议。
7. 点击“查看建议”，页面会自动跳转到详细测算结果。

手机端底部会固定显示：

- 建议方向。
- 三年净变化金额。
- “查看建议”按钮。

这样在填写参数时，不需要频繁上下滚动也能看到测算结论。

## 十、常见问题

### 1. 为什么输入报销金额后建议会变？

因为报销金额会改变支缴率。支缴率越高，未来上浮概率越高，三年额外保费也可能更高。

### 2. 为什么小额报销也可能建议内部解决？

如果企业当前享受 50% 下浮优惠，哪怕报销金额不大，也可能清零连续无理赔记录，导致未来几年失去优惠。工具会把这部分机会成本计入三年测算。

### 3. 为什么有时建议正常报销？

如果本次可报销金额明显高于未来三年额外保费，工具会建议正常走工伤报销程序。

### 4. 一类行业为什么选项不同？

一类行业费率较低，且规则中下浮逻辑与二至八类不同，因此页面会自动切换一类行业的状态选项。

### 5. 这个结果能直接作为最终决策吗？

不建议。它适合作为内部测算依据。最终还应结合：

- 当地社保经办机构核定。
- 工伤事实和法律责任。
- 企业合规要求。
- 员工权益保障。
- 安全生产管理要求。
